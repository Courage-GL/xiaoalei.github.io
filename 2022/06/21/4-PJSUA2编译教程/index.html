4.PJSUA2编译教程

发表于2022-06-21|更新于2022-06-21
|阅读量:

1.下载资源

1.1 下载openssl

git clone git://git.openssl.org/openssl.git 下载opensll文件

1.2 下载pjproject-2.12.1

2.准备编译

2.1 修改相关配置 在config_site.h文件下面修改相关配置

/* Activate Android specific settings in the ‘config_site_sample.h’ */
// 指定编译平台为Android
#define PJ_CONFIG_ANDROID 1
#include <pj/config_site_sample.h>
#define PJMEDIA_HAS_VIDEO 1
#define PJ_HAS_IPV6 1
// 使用ssl
#define PJ_HAS_SSL_SOCK 1
#define PJSIP_HAS_TLS_TRANSPORT 1
// 心跳 300
#define PJSIP_TCP_KEEP_ALIVE_INTERVAL 300
// 语音降噪算法相关
#define PJMEDIA_WEBRTC_AEC_USE_MOBILE 1
#define PJMEDIA_HAS_WEBRTC_AEC 1

2.2 编译openssl文件 根据android不同平台生成不同的openssl文件

编译命令如下：

1.git clone git://git.openssl.org/openssl.git 下载opensll文件
2.cd进入到openssl目录下 编译源码
3.使用export命令新增/修改环境变量ANDROID_NDK_ROOT指向你的NDK目录，export 的效力仅限于该次登录操作exportANDROID_NDK_ROOT=/Users/gaolei/Library/Android/sdk/ndk/22.0.7026061
4.将编译openssl所需的NDK命令所在的目录添加到环境变量PATH中PATH=$ANDROID_NDK_ROOT/toolchains/llvm/prebuilt/darwin-x86_64/bin:$PATH
5.运行./Configure脚本生成Makefile文件，这里指定生成arm64的.so// 指定编译openssl输出位置 生成opensslso文件./Configure android-arm64 -D__ANDROID_API__=21 –prefix=/Users/gaolei/Desktop/opensslarm64./Configure android-arm -D__ANDROID_API__=21 –prefix=/Users/gaolei/Desktop/opensslarm./Configure android-x86 -D__ANDROID_API__=21 –prefix=/Users/gaolei/Desktop/opensslx86 ./Configure android-x86_64 -D__ANDROID_API__=21 –prefix=/Users/gaolei/Desktop/opensslx86_64执行完之后会在当前目录下生成一下两个文件 makeFile
6.使用make && make install 命令进行编译(armeabi 对应 android-arm, arm64-v8a 对应 android-arm64, x86 对应 android-x86 ，x86_64对应 android-x86_64)
7.执行完会在桌面生成想对应的android平台的openssl文件夹

2.3 编译pjsua2文件

1.cd /Users/gaolei/Downloads/pjproject2.12.1 #cd到源码根目录 make clean
2.cd pjsip-apps/src/pjsua/android/jni make clean
3.cd /Users/gaolei/Downloads/pjproject-2.12.1/pjsip-apps/src/swig make clean
4.cd /Users/gaolei/Downloads/pjproject-2.12.1 export ANDROID_NDK_ROOT=/Users/gaolei/Library/Android/sdk/ndk/22.0.7026061
5.编译不同平台的命令

APP_PLATFORM=android-20 TARGET_ABI=arm64-v8a ./configure-android –use-ndk-cflags –with-ssl=/Users/gaolei/Desktop/opensslarm64

APP_PLATFORM=android-20 TARGET_ABI=armeabi-v7a ./configure-android –use-ndk-cflags –with-ssl=/Users/gaolei/Desktop/opensslarm

APP_PLATFORM=android-20 TARGET_ABI=x86 ./configure-android –use-ndk-cflags –with-ssl=/Users/gaolei/Desktop/opensslx86

APP_PLATFORM=android-20 TARGET_ABI=x86_64 ./configure-android –use-ndk-cflags –with-ssl=/Users/gaolei/Desktop/opensslx86_64

make dep && make clean && make

6.cd pjsip-apps/src/swig make

每次执行如上步骤，最后生成一套cpu架构的库，位于
pjsip-apps/src/swig/java/android/pjsua2/jniLibs,可以先复制出来，因为make clean之后会删除掉，这样如上步骤如此反复，即可生成多套库。

3.将编译生成的.so文件放到项目里 编译运行 修改相关代码 能跑起来 登录发消息都正常 意味着编译成功
